# 实验记录与改进思路

> 本文档记录 1D U-Net 基线版本（`v0.1.0-baseline`）的完整训练过程、遇到的问题、
> 修复方案和后续优化建议，供下一轮实验参考。

## 一、训练结果（最终基线）

### 模型配置
- 架构：1D U-Net，depth=3，base_channels=32，softmax 输出头
- 参数量：629,857
- 训练设备：CPU
- 训练轮数：300 epoch，batch_size=64，samples_per_epoch=1024
- 优化器：AdamW，lr=3e-4，余弦退火调度
- 损失：MSE + EMD（各权重 1.0），EMD 使用 index 空间（均匀 bin 距离）
- 训练能量：200/300/400/500/800/1000/4000/10000 MeV（8 个）
- 留出能量：600/2000/7000 MeV（3 个，仅用于验证/测试）

## 二、训练过程中遇到的问题与修复

### Bug：softmax 输出头与 log 压缩目标空间不匹配（致命）

**现象**：
- 训练 300 epoch，损失从 65740 只降到 65670（降幅 0.1%），几乎不动
- GCR 峰位误差 59 bin，预测谱为完全平坦的噪声
- 留出集 EMD = 67974，GCR EMD = 72.6

**根因**：
- 模型输出头为 `softmax` → 预测值为归一化概率密度，数值约 0.001～0.05，**和为 1**
- 目标谱同时做了 log 压缩：`log1p(1e4 × density)` → 数值约 3～7，**和约 1000**
- 预测与目标处于不可对齐的数值空间，EMD/MSE 均被一个无法弥合的常数偏移主导
- 梯度无效，网络躺平输出近似均匀分布

**修复方案**：
- 目标谱保持为**纯归一化密度**（不做 log 压缩），与 softmax 头输出空间一致
- log 压缩**仅作用于网络输入**（探测器 A），作为 6 个数量级动态范围的表示技巧
- 新增 `build_preprocessors(cfg)` 函数，根据输出头类型自动构造正确的 `(input_pre, target_pre)` 对
- 修改 `dataset.py`、`train.py`、`run_eval.py`、`run_gcr.py`、`evaluate.py` 使用分离的预处理器

**修复效果**（300 epoch 完整训练）：

| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| 训练损失（最终） | 65670（不动） | **0.165** |
| 验证损失（最优） | 68300 | **0.430** |
| GCR EMD | 72.6 | **7.33** |
| GCR 峰位误差 | 59 bin | **2 bin** |
| 留出集尾部保真度 | 0.028 | **0.975** |

## 三、当前结果的局限性

### 1. 预测谱在远离峰区有约 1e-5 的平底

**表现**：图中红色预测线在 10～1000 keV 区间没有完全跌落，维持约 1e-5～1e-6 的水平，
而真实 TEPC 谱在该区间快速衰减。

**原因**：softmax 头无法输出真正的零——360 个 bin 的概率总和必须为 1，
即使峰区以外的区域应趋近于零，也会被均摊一点点概率质量。

**影响**：EMD 的主要贡献来自这个平底（CDF 在高能区收敛速度稍慢）。

### 2. CDF 曲线在 1～3 keV 处预测略超前（质量偏低能端）

**表现**：CDF 图中红虚线比绿实线在低能侧稍快上升，说明预测谱的质心比真实谱略偏低能。

**原因**：GCR 宽谱的低能端比任何单能训练谱都更"软"（更多低能沉积），
合成训练数据的 `gcr_like` 幂律权重覆盖不够充分。

## 四、后续优化建议

按优先级从高到低排列。

### 优先级 A：直接可调，效果确定

#### A1. 加大 EMD 权重

```yaml
# configs/baseline.yaml
loss:
  w_mse: 0.5
  w_emd: 2.0
```

EMD 直接优化谱形整体对齐；加大权重可压制平底并改善 CDF 曲线重合度。
MSE 权重适当降低以保持总损失量级稳定。

#### A2. 提高 gcr_like 训练样本比例

```yaml
synth:
  mixture_families: [mono, sparse, dense, gcr_like, gcr_like]
```

让 `gcr_like`（幂律权重宽谱）在随机采样中出现概率从 25% 提升到 40%，
使训练分布更接近真实 GCR 形状，改善低能端偏移。

#### A3. 使用对数能量空间的 EMD 距离

```yaml
loss:
  emd_space: log_energy
```

当前默认 `index`（bin 索引，均匀权重）。改为 `log_energy` 后，
EMD 的 bin 距离 $d_i = \Delta(\log_{10} E)$ 为常数（因为网格是等比的），
物理上更自然——每个数量级的形状误差被同等惩罚，而不是低能密集 bin 主导。

#### A4. 增加训练轮数或样本量

当前验证损失在 ~170 epoch 达到最优后有轻微反弹，说明模型容量足够但样本多样性略不足。
可尝试：

```yaml
train:
  epochs: 500
  samples_per_epoch: 2048
```

### 优先级 B：架构改进，需重新训练

#### B1. 加深 U-Net（depth=4）

```yaml
model:
  depth: 4
  base_channels: 32
```

depth=4 时感受野覆盖整个 360 bin，对宽谱的全局形状建模能力更强。
360 = 2^3 × 45，第 4 层步长后长度为 22，需注意 pad/crop，代码已支持。

#### B2. 增加 base_channels

```yaml
model:
  base_channels: 64  # 原为 32
```

参数量约 4 倍（~250 万），增强表达能力。对只有 360 bin 的 1D 信号，
这仍是轻量级网络，CPU 训练可接受。

#### B3. 输入加入一次束流能量辅助通道（Section 13 预留功能）

将已知的原始能量（单能训练时已知，GCR 时可用 GCR 注量谱的质心估计）
作为第 2 个输入通道，帮助模型区分不同能量下的响应差异。

```yaml
model:
  in_ch: 2  # 原为 1
```

需在 `dataset.py` 里为输入拼接归一化能量通道。

### 优先级 C：数据策略改进

#### C1. 调整留出能量

当前留出 600/2000/7000 MeV。可尝试改为：

```yaml
data:
  heldout_energies_MeV: [400, 1000, 4000]
```

测试不同插值难度下的泛化能力，确认结论的稳健性。

#### C2. Poisson 噪声强度可配置

当前 Poisson 噪声水平为随机 1e3～1e5 counts，可以加一个配置项：

```yaml
synth:
  poisson_counts_range: [1000, 100000]
```

并测试关闭噪声（`poisson_noise: false`）时 GCR 指标是否更好，
以判断噪声是在帮助正则化还是在引入不必要的难度。

#### C3. GCR 功率律指数调优

```yaml
synth:
  gcr_powerlaw_index: -2.7  # 标准值；可尝试 -2.5 到 -3.0
```

实际 GCR 质子能谱在 GeV 量级的谱指数约为 -2.7，但低能段受太阳调制，
谱变软（更多低能质子）。适当调软（如 -3.0）可能改善低能端偏移。

## 五、下一步实验计划（推荐顺序）

```
第1轮：A1 + A2（调权重 + 提高 gcr_like 比例）
  → 观察 GCR EMD 是否从 7.33 降到 5 以下，CDF 曲线是否更重合

第2轮：A3（log_energy EMD 空间）
  → 对比 index vs log_energy 的 GCR 图，判断物理距离权重是否有帮助

第3轮：B1（depth=4）
  → 更深网络 + A 组最优配置

第4轮：B3（能量辅助通道）
  → 单能训练时直接提供能量信息，测试是否改善留出集泛化
```

## 七、重要设计决策备忘

1. **softmax 头 + 纯密度目标**：两者必须配套，log 压缩只能用于输入端。
   若未来换用线性输出头（`head: linear`），则目标可以做 log 压缩，
   届时需相应修改 `build_preprocessors()` 中的逻辑。

2. **留出整个能量而非随机划分样本**：这是测试"对未见能量插值"能力的唯一正确方式。
   若只随机划分样本，验证集和训练集来自相同能量，无法检验真正的泛化。

3. **GCR 谱不参与训练，也不参与合成混合**：它是最终测试集，任何形式的泄露都会
   导致结论不可信。代码中 `GCR_spectrum.xlsx` 只在 `run_gcr.py` 中被加载。

4. **响应线性假设**：本项目的合成数据策略的核心物理假设是探测器响应对注量线性叠加。
   当前 GCR 测试结果（峰位误差 2 bin）表明该假设在本仿真数据中成立。
   若使用真实测量数据，需重新验证此假设。
